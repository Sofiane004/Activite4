À quoi sert concrètement un système de gestion de version ?

Il aborde l'utilité d'un système de gestion de version, en mettant en évidence les défis rencontrés lors de la collaboration sur un projet et la nécessité d'avoir un historique des modifications. Il explique comment Git facilite la coordination du travail entre plusieurs personnes en conservant un historique des changements effectués sur les fichiers.

Les deux modèles des logiciels de gestion de version

il poursuit en discutant des deux modèles de gestion de version : le modèle centralisé et le modèle décentralisé. Git est présenté comme un modèle décentralisé, offrant des avantages tels que la simplicité, la flexibilité du travail et la sécurité des données.

Qu’est-ce que GitHub ?

Git est un logiciel de gestion de version. Git va nous permettre d’enregistrer les différentes modifications effectuées sur un projet et de pouvoir retourner à une version précédente du projet. GitHub est un service en ligne qui permet d’héberger des dépôts ou repo Git. C’est le plus grand hébergeur de dépôts Git du monde.

Installation de Git

La façon la plus simple d’installer Git est de télécharger la dernière version sur le site officiel ICI, d’ouvrir le fichier téléchargé et de suivre les instructions à l’écran en laissant toutes les valeurs par défaut.

Paramétrage de Git

Le paramétrage de Git permet de configurer les préférences et le comportement de Git sur votre système. Il existe un paramétrage global pour tous les projets et un paramétrage local spécifique à chaque projet. Les configurations courantes incluent le nom d'utilisateur, l'adresse e-mail et d'autres options personnalisées. Le paramétrage de Git se fait à l'aide des commandes "git config --global" pour le paramétrage global et "git config" pour le paramétrage local.

La gestion des informations selon Git

La façon dont Git gère les informations tel qu'Il utilise des instantanés pour enregistrer les changements, permet la création de branches pour travailler parallèlement sur différentes fonctionnalités, et offre des fonctionnalités avancées pour faciliter la collaboration et la gestion des versions du projet.

Les états des fichiers

Git gère les fichiers dans trois principaux états : modifié, indexé et validé. Les modifications apportées aux fichiers sont d'abord marquées comme modifiées, puis sélectionnées pour l'indexation et enfin validées pour créer des instantanés permanents. La gestion des états des fichiers dans Git permet un suivi précis des modifications et facilite la collaboration et la récupération des versions antérieures.

Les zones de travail

Les états de fichiers sont liés à des zones de travail dans Git. En fonction de son état, un fichier va pouvoir apparaitre dans telle ou telle zone de travail. Tout projet Git est composé de trois sections : le répertoire de travail (working tree), la zone d’index (staging area) et le répertoire Git (repository). Le répertoire de travail ou “working tree” correspond à une extraction unique (“checkout”) d’une version du projet. Les fichiers sont extraits de la base de données compressée située dans le répertoire Git et sont placés sur le disque afin qu’on puisse les utiliser ou les modifier. La zone d’index ou “staging area” correspond à un simple fichier, généralement situé dans le répertoire Git, qui stocke les informations concernant ce qui fera partie du prochain instantané ou du prochain “commit”.

Créer un dépôt Git à partir d’un répertoire existant

Pour créer un dépôt Git à partir d'un répertoire existant :

  1. Assurez-vous d'avoir Git installé.
  2. Entrez dans le répertoire du projet dans le terminal.
  3. Utilisez `git init` pour initialiser le dépôt Git.
  4. Utilisez `git add` pour ajouter les fichiers au dépôt.
  5. Utilisez `git commit` pour valider les modifications avec un message de commit.
  6. Facultatif : configurez un dépôt distant et poussez vos commits vers ce dépôt.
